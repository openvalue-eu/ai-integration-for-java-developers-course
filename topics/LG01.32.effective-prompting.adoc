=== Comment-Driven Development
[.notes]
--
**Instructor Notes:** Demonstrate live. Show how comment quality affects suggestion quality.
--

**Key Principles:**
[%step]
--
- Write clear, specific comments describing intent
- Include input/output examples
- Specify error handling requirements
- Mention framework/library preferences (Spring, JPA, etc.)
--
**Example:**
```
// Create a method to calculate compound interest
// Parameters: principal (double), rate (double), time (int), frequency (int)
// Returns: final amount as BigDecimal
// Handle negative values by throwing IllegalArgumentException
```

=== Prompt Patterns for Java Development
[.notes]
--
**Instructor Notes:** Provide template patterns participants can reuse. Show before/after examples.
--

**Common Patterns:**

**CRUD Operations:**
```
// Create a JPA repository method to find all users by email domain
// Return List<User>, parameter: String emailDomain
```

**REST Controllers:**
```
// Create Spring REST controller endpoint for POST /api/orders
// Accepts OrderRequest DTO, returns OrderResponse
// Include validation and error handling
```

**Service Layer:**
```
// Implement service method to process payment transaction
// Use PaymentGateway interface, handle timeout exceptions
// Log all transactions, return TransactionResult
```

=== Iterative Refinement
[.notes]
--
**Instructor Notes:** Demonstrate accepting partial suggestion, then refining with additional prompts.
--

**Key Technique:**
[%step]
--
1. Accept initial Copilot suggestion
2. Add refinement comment above
3. Let Copilot adjust implementation
4. Repeat until satisfactory
--
**Example Refinement:**
```
// Also add caching using Spring @Cacheable
// Cache key should be userId + dateRange
```

[%step]
--
